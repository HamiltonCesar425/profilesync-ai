# ProfileSync AI — Relatório de Iteração

**Data:** 01/07/2026

**Iteração:** 04

---

## Objetivo da iteração

Implementar o módulo de **Experiência Profissional**, permitindo registrar o histórico profissional do usuário de forma estruturada, vinculada ao perfil autenticado e preparada para futura integração com a exportação ATS.

---

## Funcionalidades implementadas

## Modelagem de Dados

Criada a entidade **ProfessionalExperienceModel** contendo:

- vínculo ao Profile;
- empresa;
- cargo;
- tipo de contratação;
- modalidade de trabalho;
- localização;
- descrição das atividades;
- data de início;
- data de término;
- indicador de experiência atual;
- timestamps de criação e atualização.

Também foi adicionada a relação bidirecional entre **ProfileModel** e **ProfessionalExperienceModel**, utilizando `relationship()` com `cascade="all, delete-orphan"`.

---

## Schemas

Implementados os contratos Pydantic:

- ProfessionalExperienceCreate
- ProfessionalExperienceUpdate
- ProfessionalExperienceResponse

Os schemas seguem o padrão arquitetural já utilizado pelas entidades Profile, Resume, Project e Technology.

---

## Repository

Implementado o **ProfessionalExperienceRepository** com operações de persistência:

- create
- list_by_profile_id
- get_by_id
- get_by_id_and_profile_id
- update
- delete

As consultas foram limitadas ao escopo do Profile, preservando o isolamento entre usuários.

---

## Service

Implementado o **ProfessionalExperienceService** contendo as regras de negócio da funcionalidade.

Principais responsabilidades:

- validar existência do Profile;
- validar posse do Profile pelo usuário autenticado;
- criar experiências profissionais;
- listar experiências;
- consultar experiência específica;
- atualizar experiências;
- remover experiências.

Toda a lógica permaneceu concentrada na camada de serviço, mantendo as rotas enxutas.

---

## API REST

Implementadas rotas autenticadas:

- POST /profiles/{profile_id}/experiences
- GET /profiles/{profile_id}/experiences
- GET /profiles/{profile_id}/experiences/{experience_id}
- PUT /profiles/{profile_id}/experiences/{experience_id}
- DELETE /profiles/{profile_id}/experiences/{experience_id}

As rotas seguem o mesmo padrão arquitetural utilizado nos módulos de Projects, Technologies e Resumes.
