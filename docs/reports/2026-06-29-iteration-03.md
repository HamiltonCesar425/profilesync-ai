# Iteration Report — 2026-06-29

## Objetivo

Implementar o gerenciamento completo de Projetos (Projects), permitindo que usuários autenticados cadastrem e mantenham projetos vinculados exclusivamente aos seus próprios perfis profissionais.

---

## Funcionalidades implementadas

### Modelo de domínio

* Criação da entidade `ProjectModel`.
* Relacionamento com `ProfileModel`.
* Suporte à associação de tecnologias.
* Correções de nomenclatura (`start_date`, `updated_at`).

---

### Camada Repository

Implementado o `ProjectRepository` contendo:

* create()
* list_by_profile()
* get_by_id()
* update()
* delete()

A persistência passou a utilizar:

```python
model_dump(mode="json")
```

garantindo serialização correta de tipos especiais do Pydantic, como `HttpUrl` e datas.

---

### Camada Service

Implementado o `ProjectService` contendo todas as regras de negócio.

Principais validações:

* propriedade do perfil;
* existência das tecnologias informadas;
* isolamento entre usuários;
* atualização segura;
* exclusão segura.

---

### Camada API

Implementadas rotas autenticadas para:

* POST /projects
* GET /projects/profile/{profile_id}
* GET /projects/{project_id}
* PUT /projects/{project_id}
* DELETE /projects/{project_id}

Todas protegidas por JWT.
